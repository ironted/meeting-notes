Using Materials in FireMonkey 3D

Objects3D Hierarchy


MaterialSources


ColorMaterialSource

  • Color - Doesn't show 3D surfaces well. Useful for drawing primitives.

TextureMaterialSource

  • Texture - Any supported image type
  • BitmapAnimation - Animate between two images
  • BitmapListAnimation - Animate through multiple frames in the same image (like sprites)

LightMaterialSource

  • Ambient (shadow/indirect)
  • Diffuse (reflected colour under direct white light)
  • Emissive (self-illuminated colour)
  • Specular (shiny spot)
  • Texture - just like TextureMaterialSource

Light

Prerequisite for LightMaterialSource

  • Color
  • Directional - Light comes from one direction, like the sun
  • Point - Light originates from TLight and shines in all directions
  • Spot - Light originates from TLight in a specific direction. Angle controlled by SpotCutOff.

MeshData

Text Properties (slow):

  • Points
  • TriangleIndices
  • TexCoordinates
  • Normals


Direct Access (preferred):

  • VertexBuffer
    • Vertices (Point3D)
    • TexCoord (PointF)
    • Normals (3D Vector)
  • IndexBuffer

Texture Mapping

Also called UV mapping (nothing to do with ultraviolet). U and V represent the X and Y axes of the texture image. Different letters are used to avoid any confusion with the X, Y and Z axes in the 3D space.

Just like with the 3D objects themselves, it's all about the triangles.

Cube Map

With the default behaviour, cubes show the entire texture image on each face.

Getting a texture to wrap around a cube requires a little tinkering.
